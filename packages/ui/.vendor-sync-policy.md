# @bfeai/ui Vendor-Sync Policy (Wave 2+)

This package is vendored across 5 repos: dashboard.bfeai (canonical),
keywords.bfeai, labs.bfeai, sma.bfeai, offpage.bfeai. Each consumer app has
its own copy of `packages/ui/`.

## Vendor THESE files when syncing dashboard → consumer apps

- `src/**/*` (all source files: components, hooks, lib, credits, auth, etc.)

## Do NOT vendor

- `package.json` `devDependencies` field — consumer apps own their test
  toolchain. dashboard's packages/ui uses vitest 4.x; consumer apps may use
  vitest 1.x or different versions. Mixing causes peer-dep conflicts.
- `tests/**/*` — modal/component tests live canonically in dashboard's
  `packages/ui/tests/`. If a consumer app needs to test ui code, it should
  test integration via its own test harness, not the package itself.
- `vitest.config.ts` — dashboard-canonical only.
- `package-lock.json` — never vendor lock files; consumer app's npm install
  generates its own.

## DO vendor (when source files change)

- `package.json` `peerDependencies` field if it changes (e.g., adding
  `@radix-ui/react-foo` to peers). Consumer apps need matching peer-dep
  entries in their root `package.json`.
- `package.json` `dependencies` field if it changes (e.g., adding a new
  runtime dep). Consumer apps need to install the new dep.

## Section D5 Wave 2 vendor scope

Three new files to copy from dashboard's `packages/ui/src/` to each consumer
app's `packages/ui/src/`:
- `lib/topup-packs.ts`
- `credits/CreditsProvider.tsx`
- `credits/OutOfCreditsModal.tsx`

Plus update `src/index.ts` in each consumer app's packages/ui to add the 3
new exports. NO changes to `package.json` (no new runtime deps), NO
test/config files.
